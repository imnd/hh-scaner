# ТЗ: Приложение для точечного поиска вакансий и прямых откликов

## 1. Назначение

Инструмент для персонального использования соискателем. Ищет вакансии на hh.ru по ключевым словам, собирает публично доступные контактные данные компаний/HR, и помогает составить и отправить персонализированный отклик по каждой конкретной вакансии — вручную, с подтверждением пользователя на каждом шаге.

**Принципиально:** приложение НЕ делает массовую автоматическую рассылку. Отправка — только по явному действию пользователя на конкретную вакансию, после просмотра и правки текста. Это инструмент "ассистент", а не "спам-бот".

---

## 2. Функциональные модули

### 2.1. Поиск вакансий
- Ввод ключевых слов, региона, фильтров (зарплата, опыт, формат работы)
- Парсинг выдачи hh.ru (Playwright), постраничный сбор результатов
- Дедупликация по ID вакансии
- Сохранение в локальную БД: заголовок, ссылка, компания, зарплата, краткое описание, дата публикации

### 2.2. Сбор контактных данных
Для каждой вакансии, отмеченной пользователем как интересная:
- Переход на страницу вакансии → извлечение прямых контактов, если указаны рекрутером явно (данные, публично размещённые самим работодателем)
- Переход в профиль компании на hh.ru → сайт компании, соцсети (если указаны)
- Опционально: переход на сайт компании → эвристический поиск страницы "Карьера"/"Контакты" → извлечение email/формы (если общедоступны и не за формой логина)
- **Ограничение:** только данные, добровольно опубликованные самим работодателем в открытом доступе. Никакого поиска личных данных конкретных физлиц через сторонние базы, слив баз, whois-догадки и т.п.

### 2.3. Хранилище резюме
- Загрузка резюме пользователя (PDF/DOCX) — одно или несколько версий (под разные направления)
- Парсинг текста резюме для использования как контекста при генерации отклика
- Хранится локально, не передаётся никуда, кроме шага генерации отклика (см. 2.4)

### 2.4. Генерация отклика
- На основе текста вакансии + резюме пользователя формируется черновик сопроводительного письма (через LLM API или локальную LLM, см. раздел 8)
- Письмо открывается в редакторе, пользователь может (но не обязан) отредактировать текст перед отправкой

### 2.5. Отправка
- Если есть email — отправка через SMTP (данные пользователя, его собственный почтовый аккаунт)
- Если есть только форма на сайте компании — приложение просто открывает эту форму в браузере с предзаполненным текстом (без автоматического сабмита)
- Лог отправленных откликов: дата, вакансия, контакт, статус

### 2.6. Дашборд / список результатов
Таблица: вакансия → ссылка → компания → найденный контакт → статус (не обработано / отклик составлен / отправлено / ответ получен)

---

## 3. Нефункциональные требования

- **Rate limiting**: между запросами к hh.ru — случайная задержка 2–6 сек, не более 3–5 параллельных запросов без прокси
- **Обработка капчи**: при обнаружении — пауза и уведомление пользователя, никакого автообхода через сторонние капча-сервисы
- **Хранение данных**: локальная БД (SQLite), без передачи третьим лицам
- **Приватность резюме**: файл резюме не логируется, не сохраняется на внешних серверах кроме momentарной передачи в LLM API для генерации текста

---

## 4. Технологический стек (предложение)

| Компонент | Технология |
|---|---|
| Парсинг | Node.js + Playwright (Chromium) |
| Backend | Node.js (Express/Fastify) |
| БД | SQLite |
| Frontend | Svelte |
| Генерация текста | Локальная LLM (Qwen2.5 через Ollama, HTTP API) |
| Отправка почты | SMTP (через аккаунт пользователя, OAuth для Gmail/Yandex) |

---

## 5. Пользовательский флоу

1. Пользователь вводит ключевые слова → получает список вакансий
2. Отмечает интересные → приложение подтягивает контакты компании
3. Для каждой вакансии с найденным контактом — кнопка "Составить отклик"
4. LLM генерирует черновик на основе резюме + текста вакансии
5. Пользователь опционально редактирует текст в интерфейсе
6. Нажимает "Отправить" — письмо уходит через SMTP пользователя, либо открывается форма сайта компании
7. Статус вакансии обновляется в таблице

---

## 6. Ограничения и что сознательно НЕ реализуется

- Нет функции "отправить всем найденным контактам одним кликом"
- Нет сбора персональных данных конкретных сотрудников (личные телефоны, соцсети физлиц) — только то, что компания сама публикует как контакт для соискателей
- Нет автообхода капчи через сторонние сервисы
- Нет функции масштабного парсинга (тысячи вакансий за раз) — инструмент заточен под точечный, осознанный поиск, а не под лидогенерацию

---

## 7. MVP (минимальная версия для старта)

Для первой версии достаточно:
- Скрипт парсинга по ключевому слову → CSV/JSON со списком вакансий + контактами
- Отдельный скрипт: вакансия + резюме → черновик отклика (текстовый файл)
- Отправка — вручную пользователем (просто копипаст в почтовый клиент)

Автоматизацию отправки и красивый UI имеет смысл добавлять после того, как ручной процесс на MVP подтвердит, что подход в принципе работает и даёт отклики от работодателей.

## 8. Локальная LLM вместо внешнего API

Для приватности резюме и отсутствия лимитов/затрат генерацию откликов можно вынести на локальную модель вместо Anthropic API.

### 8.1. Выбор модели (без дообучения)

| Модель | Размер | Комментарий |
|---|---|---|
| Qwen2.5-14B-Instruct | 14B | Лучший вариант для русского языка из открытых моделей на сегодня |
| Qwen2.5-7B-Instruct | 7B | Компромисс скорость/качество, тянется на 8–12GB VRAM |
| saiga (на базе Llama/Mistral) | 7–8B | Дообучена под русский, но может уступать Qwen в деловых текстах |
| Mistral-Nemo-Instruct | 12B | Неплохой русский, хорошо следует инструкциям |

Рекомендация для старта: **Qwen2.5-14B-Instruct** (или 7B при слабом железе) через **Ollama** или **LM Studio** — запуск без дообучения, few-shot промпт с 2–3 примерами хороших писем.

### 8.2. Дообучение (опционально, если нужен свой стиль)

Имеет смысл только при наличии 50+ реальных примеров писем пользователя. Метод — LoRA/QLoRA поверх базовой модели, тянется на потребительской видеокарте.

Инструменты: **Unsloth** (проще и быстрее для старта), **Axolotl** (более гибкий), **LLaMA-Factory** (готовый UI).

Формат обучающих данных:
```json
{
  "instruction": "Напиши сопроводительное письмо на основе вакансии и резюме",
  "input": "Вакансия: [текст]\nРезюме: [текст]",
  "output": "[реальный отклик пользователя]"
}
```

### 8.3. Интеграция в архитектуру

Node.js backend обращается к Ollama через его локальный HTTP API (обычно `http://localhost:11434`) — обычный fetch-запрос, без дополнительных Python-обвязок. Уже отражено в стеке технологий (раздел 4).
